# week-billioner-bot

24/7 бот для мониторинга Polymarket + мировых событий с приоритетом крипто-новостей.

## Что делает бот

- Собирает новости из всех RSS-источников в `WORLD_FEEDS` и события Polymarket.
- Приоритизирует крипто-триггеры и макро/геополитику (Украина, Ближний Восток, санкции, ставки ФРС и т.д.).
- Особый приоритет: Bitcoin (BTC), особенно рынки на рост/падение (golden BTC сигналы).
- Отправляет событие на анализ 1 LLM (ChatGPT 5.3).
- Работает в информ-режиме: отправляет сигналы и ссылки на новости, без live-исполнения ордеров.
- В ручном режиме отправляет actionable-сигналы с кнопками:
  - `Отклонить`
  - `Принять ставку`
  - При подтверждении бот повторно проверяет актуальность рынка и только затем отправляет ордер в Polymarket.
- Ведет статистику: победы/поражения, PnL, стартовый vs текущий портфель, ожидающие решения.

## Админ-панель (Telegram, на русском)

Поддерживается приватный доступ по `ADMIN_TELEGRAM_USER_ID`.

Команды:

- `/start` или `/help` — помощь и меню.
- `/panel` — кнопочная панель:
  - `Статистика`
  - `Текущие ставки`
  - `История ставок`
  - `Ожидают решения`
  - `Переключить авто/ручной режим`
  - `Настройки и LLM`
- `/llm` — мастер настройки единственной LLM через чат (сохранение в runtime config).
- `/pm` — мастер привязки Polymarket (ключ + funder) через чат.
- `/status` — статус runtime.
- `/keys` — список ключей.
- `/set KEY VALUE` — изменить настройку без рестарта.
- `/show KEY` — посмотреть значение (секреты маскируются).
- `/mode auto` — автоисполнение.
- `/mode manual` — ручное подтверждение.
- `/settle ID win|loss [pnl_usd]` — вручную закрыть ставку для учета статистики.

## Быстрый запуск

1. Скопируй конфиг:
```bash
cp .env.example .env
```

2. Заполни `.env`:
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`
- `ADMIN_TELEGRAM_BOT_TOKEN`
- `ADMIN_TELEGRAM_USER_ID`
- `LLM_1_*` (основная единственная LLM)

3. Запуск:
```bash
docker compose up -d --build
```

4. Проверка:
```bash
curl http://localhost:8000/health
curl -X POST http://localhost:8000/run-once
```

## Ключевые параметры в `.env`

- `AUTO_EXECUTE` — автоисполнение без ручного подтверждения (`true`) или ручной режим (`false`).
- `DRY_RUN` — симуляция исполнения без отправки реального ордера.
- `POLYMARKET_*` — параметры привязки аккаунта для live-ставок после подтверждения.
- `MAX_BET_USD`
- `MIN_CONFIDENCE`
- `MAX_DAILY_LOSS_USD`
- `MIN_MARKET_VOLUME`
- `INITIAL_BANKROLL_USD`
- `USER_LANGUAGE` (`ru`/`en`, русский по умолчанию)
